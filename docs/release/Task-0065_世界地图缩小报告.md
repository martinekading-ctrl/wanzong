# Task-0065 世界地图缩小报告

## 规格

- 旧地图：384×384 格，6144×6144 世界坐标，147456 个地形格。
- 新地图：272×272 格，4352×4352 世界坐标，73984 个地形格。
- 新地图面积约为旧地图的 50.17%，Tile 仍为 16×16，未裁剪或缩放旧地图。

## 实施

- `WorldMapSpec.gd` 是唯一地图规格来源；预览、烘焙、正式地图、相机、领地与测试均读取该规格。
- `WorldMapAnchors.gd` 是宗门、26 个资源点与 6 个建设点的唯一逻辑锚点来源；锚点先归一化，再映射到当前规格。
- 大陆与五座外岛继续使用归一化布局；噪声频率按 384/272 补偿，以保持细节密度。
- 宗门与资源点改为归一化锚点；正式地图加载后按稳定 ID 顺序统一落到安全陆地，并以 16 格为最小资源间距规则。
- `WorldResourceBaseline.gd` 逐条验证原有 26 个资源点的 ID、名称、类型、等级、储量和归属；仅位置适配紧凑地图。
- 存档世界数据增加 `world_map_layout_version`。旧布局仅迁移宗门、资源、建设点、领地和战役目标等明确地图坐标，不缩放其他 Vector2。

## 烘焙记录

- 使用 `BakeWorldMapCLI.gd` 调用事务式 `WorldMapBaker` 重建正式资源。
- 本地最新烘焙：地形生成约 32.8 秒，自然物收集约 2.2 秒，完整烘焙约 42.5 秒。
- 烘焙结束后 staging 目录为空；正式运行不调用预览生成器或烘焙器。
- 烘焙在 marker 校验、临时资源保存、场景验证或事务提交任一步失败时，都会统一清理 staging；发布检查会递归拒绝 `.tmp`、`.bak` 及非空 staging 目录。
