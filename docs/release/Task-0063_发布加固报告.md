## 修改摘要

### 全局音量设置

新增 `SettingsManager`，将主音量、音乐音量、效果音保存到 `user://settings.cfg`。`GameState.new_game()` 重置世界状态时不会再影响用户音量。旧存档内的 `game_settings` 继续读取和保存以保持兼容，但不再覆盖当前用户全局音量。

### 继续游戏存档回退

`SaveManager` 现在按修改时间检查所有存档，并通过只读解析、迁移与验证选择最近有效存档。损坏、未来版本或不兼容存档会被跳过并产生 warning；检查过程不调用 `apply_snapshot()`，不会污染当前游戏状态。主菜单会提示跳过数量。

### 地图烘焙事务

`WorldMapBaker` 先写入临时输出目录并验证临时 TileSet、MultiMesh、场景和安全陆地数据，再通过 `WorldMapBakeTransaction` 一次提交。提交中任一写入失败会恢复原文件；临时目录会清理。正式运行路径仍仅加载 `res://scenes/world/GeneratedWorldMap.scn`，不调用完整地图生成器。

### 自动化

新增统一入口 `scripts/tools/RunAllTests.gd` 和 GitHub Actions 工作流。快速回归覆盖独立 Task 测试；十年模拟通过 `workflow_dispatch` 的 `include_slow` 显式触发。日志会以 artifact 上传。

## 无法自动替代的验收

1. 启动 `WanZong.exe`，调整三个音量后开始新游戏，确认不重置。
2. 退出并重启 EXE，确认音量仍保留。
3. 创建多个存档，手动损坏最新自动存档，确认继续游戏回退至较早有效存档并显示提示。
4. 连续三次进入宗门并返回地图，检查地图、资源点、宗门、相机与缩放。
5. 在集成显卡 Windows 设备上复测一次。
