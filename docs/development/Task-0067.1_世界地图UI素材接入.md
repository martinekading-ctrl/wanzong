# Task-0067.1：世界地图 UI 素材接入

## 范围

本任务只替换世界地图的 HUD 显示层。地图烘焙结果、坐标、五宗门、26 个资源点、6 个建设点、存档版本和游戏数值均未修改。

正式界面由 `res://scenes/ui/components/WorldHUD.tscn` 和 `WorldHUD.gd` 承担；`World.gd` 继续持有选择、相机、领地显示、保存和场景跳转逻辑，只向 HUD 提供真实显示数据和接收 UI 信号。

## 素材处理与清单

处理器为 `tools/ui/process_world_map_ui_assets.py`。它以编号识别源文件，而不依赖中文文件名，并执行：

- 边缘连通的近白色 flood-fill，只去除白底，不误删内部玉石和金色高光；
- 1 像素透明边缘羽化、主体裁剪及 6 像素安全留白；
- 使用邻近深色纹理采样重建可拉伸面板中心，移除样例文字、假进度条、旗帜和占位线；
- 输出 RGBA PNG，并验证尺寸、透明角与透明/不透明 alpha 像素；
- 生成 `manifest/ui_asset_manifest.json`，记录来源、原始/处理后尺寸、九宫格边距、用途和处理说明；
- 生成联系表 `docs/development/images/task_0067_world_ui_assets_contact_sheet.png`。

源图中和样例卡片紧密合成、无法无损独立提取的资源/地图图标均在 manifest 中标记为 `unavailable`。运行时使用文字按钮，不会伪造新图标。

## 九宫格与主题

现有全局主题 `res://assets/ui/wanzong_theme.tres` 已扩展为 `StyleBoxTexture` 九宫格，未创建第二套全局主题。主要变体包括：

- `WZWorldTopPanel`、`WZWorldTopResourcePanel`、`WZWorldNavigationPanel`
- `WZWorldDetailsPanel`、`WZWorldDetailsGroup`、`WZWorldBottomPanel`
- `WZPrimaryTextureButton`、`WZNavigationButton`、`WZNavigationButtonSelected`、`WZNavigationButtonDisabled`
- `WZWorldSectionTitle`、`WZWorldBody`、`WZWorldMutedText`、`WZWorldResourceText`

边框、金色角饰和玉石装饰留在九宫格边缘；可扩展中心区域不含示例文字。WorldHUD 脚本不写死颜色、图片路径、边框或九宫格参数。

## HUD 位置与响应式规则

- 顶部：玩家宗门名、游戏日期、灵石/粮食/木材/石材、保存与禁用设置按钮。
- 左侧：宗门、弟子、建设、世界、外交；世界默认选中，只有宗门调用现有宗门详情入口，其余未完成入口保持禁用并带提示。
- 右侧：未选择时显示真实世界统计；宗门、资源点、建设点选择后显示相应真实数据。资源详情绝不显示内部 `resource_id`。
- 底部：缩放、定位青玄宗、领地开关和全图，均调用现有 `WorldCamera` / `TerritoryLayer` 行为。

布局使用锚点和容器，在 1280×720、1920×1080、2560×1440 上保持边界可见；顶部、左侧、右侧和底部均保留大部分中心地图交互区域。

## 交互与视觉收口

- `TerritoryLayer` 初始隐藏，底部领地按钮只控制该层显示，不修改领地算法。
- 宗门名称默认显示并有深色描边；玩家宗门采用玉绿/金色反馈，选中环略大于图标，悬停仅轻微缩放。
- 资源点名称仅在悬停或选中时显示，保留深色描边和细选中环。
- 建设点改为古金色细框，不再是调试感黄色实心块。

## 未改动内容

未修改 `assets/generated/**`、`GeneratedWorldMap`、烘焙工具、地图规格/锚点、`WorldDataManager` 数据结构、SaveManager 存档版本、宗门/资源/建设点基线、主菜单、字体、地形美术及所有玩法规则。
