# Task-0067：Pre-Alpha 垂直切片开发基线

## 1. 当前基础

- 当前 `master` 已包含 Task-0064、Task-0065 与 Task-0066 的稳定化成果。
- 五宗门已经进入主线，世界地图与存档底层稳定；表现层仍未完成。
- 项目产品版本为 **0.5.0**，开发阶段为 **Pre-Alpha**。

## 2. 第一段玩家流程

第一段可验收流程固定为：启动游戏 → 主菜单 → 新游戏 → 青玄宗主场景 → 查看弟子 → 建造一座建筑 → 分配弟子 → 世界地图 → 查看一个 AI 宗门 → 执行一次外交 → 保存 → 重新启动 → 继续游戏。

这个流程用于验证玩家能完成一次经营决策、一次世界查看、一次基础外交和一次存档恢复；它不是正式 1.0 的全部功能承诺。

## 3. 目标平台与输入

- 当前目标平台：Windows PC。
- 标准分辨率：1920×1080。
- 最低布局测试：1280×720。
- 额外布局测试：2560×1440。
- 输入：鼠标与键盘。
- 当前不制作手机版、完整手柄适配或超宽屏专项设计。

## 4. UI 开发原则

- 继续使用全局 Theme、Anchor 与 Container，避免依赖固定绝对坐标。
- 数据逻辑必须与 UI 分离；通用弹窗和 Tooltip 必须使用统一入口。
- 缺失图标必须有 fallback，缺少素材时 UI 不得崩溃。
- 主要操作必须有清晰反馈；1280×720 不允许主要控件越界。

## 5. 场景开发原则

首阶段只聚焦主菜单、青玄宗主场景、世界地图、弟子面板、建筑面板、基础外交面板与保存读取面板。暂不制作所有建筑的内部场景。

## 6. 美术方向

- 中式修仙宗门经营，采用 2D 斜俯视或轻等距表现。
- 主色为青灰瓦、深色木构、灰白青石、墨绿植物与青玉点缀；少量使用克制古金。
- 避免高饱和手游金光与欧美魔幻混入。
- 光源统一自左上、阴影统一向右下；建筑保持薄底座。
- PNG 使用透明背景，世界地图块基础保持 16×16。

本轮不生成美术图片，也不将候选素材误当作正式资源。

## 7. 资产状态

- **placeholder**：临时占位，仅用于功能开发。
- **candidate**：候选素材，方向基本正确但尚未批准。
- **approved**：已批准进入当前垂直切片。
- **final**：已经过尺寸、风格、版权与游戏内效果确认。

所有新素材必须明确标注为上述一种状态。

## 8. 本阶段不做

- 全部建筑、全部弟子立绘与完整战斗动画。
- 大规模战争表现、天气、昼夜、手机版与 Steam 功能。
- 成就、多语言、正式 1.0 发布与全部剧情插画。

## 9. 垂直切片完成标准

- 主菜单显示 0.5.0 Pre-Alpha。
- 新游戏可进入青玄宗；至少六个核心建筑可见，至少一座可建造。
- 至少一名弟子可被分配，并可进入世界地图。
- 五宗门正确显示，至少一次基础外交可完成。
- 保存和继续游戏可用；1280×720 不越界，1920×1080 表现正常。
- 无阻塞流程错误，UI、场景与美术方向开始统一。
