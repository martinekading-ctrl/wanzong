# 《万宗》项目状态

## 当前开发阶段

**0.5.0 Pre-Alpha**

项目目前处于核心系统整合与首个垂直切片开发阶段：不是 Beta，不是 Release Candidate，也不是正式 1.0。当前不具备正式发布条件。

## 已稳定的底层基础

- Godot 4.7，当前主要目标平台为 Windows PC。
- 五个固定初始宗门：1 个玩家宗门与 4 个 AI 宗门。
- 272×272 地图、4352×4352 世界尺寸与 73984 个地形格。
- 26 个资源点和 6 个建设点的基线数据。
- 五宗门数据与资源基线，以及跨系统宗门引用校验。
- 存档引用验证与早期十宗门存档拒绝保护。
- 世界地图加载、fallback 与生成目录安全检查。
- RunAllTests、Godot CI、ReleaseChecklist、五宗门/地图/存档自动回归。

这些内容证明底层数据和自动化门禁稳定，不代表整款游戏的 UI、美术或玩家体验已经完成。

## 当前未完成内容

- 正式主菜单视觉与正式宗门主场景。
- 正式建筑表现、弟子界面完善与世界地图正式 UI。
- 外交、保存读取与统一弹窗提示的正式界面。
- 正式建筑美术、环境美术、头像/立绘及资源/物品图标。
- UI 动画、场景动画和完整音频资源。
- 新手引导与完整玩家垂直流程。
- Windows 图形人工验收及 1280×720、1920×1080、2560×1440 多分辨率人工验收。

## 下一里程碑

目标流程：启动游戏 → 主菜单 → 开始新游戏 → 进入青玄宗主场景 → 查看弟子 → 建造一座建筑 → 分配一名弟子 → 进入世界地图 → 查看其他四个宗门 → 执行一次基础外交 → 保存 → 退出 → 继续游戏。

## 版本阶段定义

- **Pre-Alpha**：核心功能仍在整合，允许占位资源，目标是打通完整玩家流程。
- **Alpha**：核心循环可完整游玩，主要 UI 和场景已经存在。
- **Beta**：不再新增大型系统，主要处理内容、平衡、性能和 Bug。
- **Release Candidate**：正式内容与美术基本完成，只处理阻塞发布的问题。

当前项目只属于 **Pre-Alpha**。

## 自动化质量门禁

- `scripts/tools/RunAllTests.gd`：默认快速回归，失败返回退出码 1。
- `.github/workflows/godot-ci.yml`：对 `master` 推送和面向 `master` 的 PR 执行资源导入、快速回归、主入口 smoke 与发布检查。
- `scripts/tools/ReleaseChecklist.gd`：校验当前版本阶段、导出配置、地图、五宗门、资源点、建设点、存档和生成目录安全性。

这些自动化测试验证的是底层稳定性；它们不表示 UI、美术和玩家体验已达到正式发布标准。
